    • Use of semantic elements:

    Article: A semantic element clearly describes its meaning to both the browser and the developer. Examples of non-semantic elements: <div> and <span> - Tells nothing about its content. Examples of semantic elements: <form>, <table>, and <article> - Clearly defines its content.

    The <aside> element defines some content aside from the content it is placed in (like a sidebar). The <aside> content should be indirectly related to the surrounding content

    Details

    Specify details that the user can open and close on demand

    The <summary> tag defines a visible heading for the <details> element. The heading can be clicked to view/hide the details. Note: The <summary> element should be the first child element of the <details> element.

    The <footer> element defines a footer for a document or section. A <footer> element typically contains:

    • authorship information
    • copyright information
    • contact information
    • sitemap
    • elated documents
    • back to top links
    You can have several <footer> elements in one document.

    The <header> element represents a container for introductory content or a set of navigational links. A <header> element typically contains:

    • one or more heading elements (<h1> - <h6>)
    • logo or icon
    • authorship information

    The <main> tag specifies the main content of a document. The content inside the <main> element should be unique to the document. It should not contain any content that is repeated across documents such as sidebars, navigation links, copyright information, site logos, and search forms. Note: There must not be more than one <main> element in a document. The <main> element must NOT be a descendant of an <article>, <aside>, <footer>, <header>, or <nav> element.

    The <mark> tag defines text that should be marked or highlighted.

    The <nav> element defines a set of navigation links. Notice that NOT all links of a document should be inside a <nav> element. The <nav> element is intended only for major blocks of navigation links. Browsers, such as screen readers for disabled users, can use this element to determine whether to omit the initial rendering of this content.

    Section

    The <section> tag defines a section in a document.

    The <time> tag defines a specific time (or datetime). The datetime attribute of this element is used translate the time into a machine-readable format so that browsers can offer to add date reminders through the user's calendar, and search engines can produce smarter search results. Note: The time element does not render as anything special in any of the major browsers.
